# Быстрая настройка удаленной базы данных

## Проблема
В файле `.env` сейчас стоит шаблонная строка подключения. Нужно заменить её на реальную.

## Решение: 3 простых шага

### Шаг 1: Создайте бесплатную базу данных (если еще нет)

**Вариант A: Supabase (рекомендуется - самый простой)**
1. Перейдите на https://supabase.com
2. Нажмите "Start your project" → "Sign up" (можно через GitHub)
3. Создайте новый проект:
   - Project name: `task-manager` (любое имя)
   - Database Password: придумайте пароль (запомните его!)
   - Region: выберите ближайший
   - Нажмите "Create new project"
4. Дождитесь создания проекта (2-3 минуты)
5. Перейдите в Settings → Database
6. Найдите "Connection string" → "URI"
7. Скопируйте строку (она выглядит так: `postgresql://postgres.xxxxx:[YOUR-PASSWORD]@aws-0-us-east-1.pooler.supabase.com:6543/postgres`)

**Вариант B: Neon (альтернатива)**
1. Перейдите на https://neon.tech
2. Нажмите "Sign Up" (можно через GitHub)
3. Создайте новый проект
4. Скопируйте Connection String из дашборда

### Шаг 2: Обновите файл .env

1. Откройте файл `.env` в корне проекта
2. Найдите строку:
   ```
   DATABASE_URL="postgresql://username:password@host:5432/database?schema=public"
   ```
3. Замените её на вашу реальную строку подключения (которую скопировали в шаге 1)

   **Пример для Supabase:**
   ```
   DATABASE_URL="postgresql://postgres.xxxxx:ваш-пароль@aws-0-us-east-1.pooler.supabase.com:6543/postgres?sslmode=require"
   ```

   **Важно:** 
   - Замените `[YOUR-PASSWORD]` на реальный пароль
   - Если в строке есть `[YOUR-PASSWORD]`, обязательно замените на пароль
   - Добавьте `?sslmode=require` в конце, если его нет

### Шаг 3: Примените миграции

Откройте терминал в папке проекта и выполните:

```bash
npx prisma migrate deploy
```

Или для разработки (создаст миграцию):

```bash
npx prisma migrate dev --name init
```

Это создаст все таблицы в вашей базе данных.

### Шаг 4: Проверьте

Запустите dev-сервер:

```bash
npm run dev
```

И попробуйте зарегистрироваться на http://localhost:3000/signup

## Если что-то не работает

### Ошибка "Can't reach database server"
- Проверьте, что строка подключения скопирована полностью
- Убедитесь, что пароль в строке подключения правильный
- Проверьте, что база данных создана и доступна

### Ошибка "SSL required"
Добавьте в конец строки подключения: `?sslmode=require`

### Ошибка миграций
Попробуйте:
```bash
npx prisma db push
```
(Это создаст таблицы без миграций, для разработки)

## Нужна помощь?

Если у вас уже есть строка подключения, просто вставьте её в `.env` вместо шаблона и выполните `npx prisma migrate deploy`.
